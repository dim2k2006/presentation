Preparing design for html

I prepared this presentation to show how we can optimize and speed up our work process.

Here I have got a list of proposals for designers on how they can help frontend developers.

If you have some suggestions or if you disagree with the information on slides please let me know.

1. One sketch file for project

1.1 One sketch file for project

Here we can see one sketch file and it stores design for all project.

There is a list of pages on the left side of the screenshot.

Each page contain design for desktop, tablet and mobile.

This approach allows us to structure and organize information.

It also allows us to interact faster and much more efficient with design files.

All changes should be implemented only in this file.

And because of it we can always have current version of project design. Everyone in team knows where it is stored and everyone can interact with it.

2. Clean layers

2.1 Clean layers

Design files should not have any extra unused layers and we must keep order in layers.

This screenshot is not from sketch, but in my opinion it shows the main idea.

Here we can see that each block of page is in it own folder (group) and I think it is good example.

2.2 Clean layers

If we talk about images which consist of multiple layers, then all this layers should be in group.

2.3 Clean layers

If we use svg in our project and if the image has some text, then it should be converted to curves.

Here we can see some text layer.

2.4 Clean layers

To convert text to curves we should right click on text layer and choose «Convert to Outlines».

2.5 Clean layers

After that we can see that our text now consists of paths.
